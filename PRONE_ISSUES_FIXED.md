# 俯卧位质控问题修复说明

## 问题 1：PF 数据缺失

### 问题描述
治疗前PF、治疗后PF、PF提升率显示为 null，无法计算。

### 原因分析
FiO2 数据存储在 `bedside` 集合中（code = 'param_FiO2'），不是在 `bGATemp` 集合中。

- **PaO2 数据**：存储在 `bGATemp` 集合的 `bedsides` 数组中（code = 'param_bg_po2'）
- **FiO2 数据**：存储在 `bedside` 集合中（code = 'param_FiO2'，值在 `strVal` 字段）

### 解决方案
修改 `proneSessionUtils.js` 中的 `enrichSessionsWithBga` 函数：

1. 从 `bGATemp` 集合获取 PaO2 数据
2. 从 `bedside` 集合获取 FiO2 数据
3. 使用新的 `extractPFRatioFromSources` 函数计算 PF 比值

### 修复后效果
- 有治疗前PF的俯卧位治疗记录：21 个
- 有治疗后PF的俯卧位治疗记录：7 个
- 有治疗前后PF的俯卧位治疗记录：5 个

## 问题 2：治疗后PF数据较少

### 问题描述
有治疗后PF的俯卧位治疗记录数量较少（7个），远少于有治疗前PF的俯卧位治疗记录（21个）。

### 原因分析
这是正常的临床情况：

1. **血气检查频率**：血气检查不是每小时都做的，通常每天做 1-2 次
2. **时间窗口限制**：治疗后血气取俯卧位结束后 4 小时内最近一次
3. **实际情况**：在俯卧位结束后 4 小时内可能没有进行血气检查

### 数据示例
```
俯卧位结束时间：2026-04-28T00:00:00.000Z
查询窗口（后 4 小时）：2026-04-28T00:00:00.000Z 至 2026-04-28T04:00:00.000Z

FiO2 数据（有）：
  1. 时间: 2026-04-28T00:00:00.000Z 值: 50
  2. 时间: 2026-04-28T01:00:00.000Z 值: 50
  3. 时间: 2026-04-28T02:00:00.000Z 值: 50
  4. 时间: 2026-04-28T03:00:00.000Z 值: 40
  5. 时间: 2026-04-28T04:00:00.000Z 值: 40

PaO2 数据（无）：
  没有找到

下一次血气检查时间：2026-04-28T08:33:00.000Z（约 8.5 小时后）
```

### 解决方案
这是正常的临床数据情况，不需要修改代码。在报表中：
- 无数据时显示"无数据"或"-"
- 在指标悬浮提示中说明数据来源和时间窗口

## 问题 5：指标悬浮提示

### 问题描述
质控指标概览下的选项需要加入鼠标悬浮提示，说明分子分母和计算逻辑。

### 解决方案
为每个指标卡片添加悬浮提示框：

1. **俯卧位实施例次**
   - 说明：统计时间范围内有效的俯卧位开始事件总数（valid=true）

2. **单次时长达标率**
   - 分子：单次俯卧位时长≥16小时的有效治疗次数
   - 分母：所有有效治疗次数
   - 达标标准：单次俯卧位持续时间≥16小时

3. **适应症符合率**
   - 分子：俯卧位开始前PF<150mmHg的治疗次数
   - 分母：有治疗前血气数据的有效治疗次数
   - 适应症标准：PF比值<150mmHg（重度ARDS）

4. **治疗有效率**
   - 分子：治疗后PF较治疗前提升≥20%的治疗次数
   - 分母：有治疗前后血气数据的有效治疗次数
   - 有效标准：(治疗后PF-治疗前PF)/治疗前PF×100%≥20%

5. **累计俯卧位时长**
   - 说明：所有有效俯卧位治疗的持续时间总和
   - 仅统计正常治疗，不包含异常数据

6. **异常数据率**
   - 分子：异常治疗次数（未闭合、时长异常、顺序异常等）
   - 分母：总治疗次数
   - 异常类型：未闭合(>24h)、时长<2h、时长>24h、顺序异常、孤立结束
