## 问题修复详情

### 问题 1：PF 数据缺失 ✅ 已修复

**原因**：FiO2 数据存储在 bedside 集合中，不是在 bGATemp 集合中

**解决方案**：
1. 修改 `proneSessionUtils.js` 中的 `enrichSessionsWithBga` 函数
2. 从 `bGATemp` 集合获取 PaO2 数据
3. 从 `bedside` 集合获取 FiO2 数据
4. 新增 `extractPFRatioFromSources` 函数计算 PF 比值

**修复效果**：
- 修复前：有治疗前PF：0，有治疗后PF：0
- 修复后：有治疗前PF：21，有治疗后PF：7

### 问题 2：前端界面风格 ✅ 已修复

3. 优化卡片和表格样式
4. 添加响应式布局

### 问题 3：指标悬浮提示 ✅ 已修复

**解决方案**：

## 数据说明

### 治疗后PF数据较少的原因

治疗后PF数据较少（7个）是正常的临床情况：

1. **血气检查频率**：血气检查不是每小时都做的，通常每天做 1-2 次
2. **时间窗口限制**：治疗后血气取俯卧位结束后 4 小时内最近一次
3. **实际情况**：在俯卧位结束后 4 小时内可能没有进行血气检查

**示例**：
```
俯卧位结束时间：2026-04-28T00:00:00.000Z
查询窗口（后 4 小时）：2026-04-28T00:00:00.000Z 至 2026-04-28T04:00:00.000Z

FiO2 数据（有）：
  1. 时间: 2026-04-28T00:00:00.000Z 值: 50
  2. 时间: 2026-04-28T01:00:00.000Z 值: 50
  3. 时间: 2026-04-28T02:00:00.000Z 值: 50
  4. 时间: 2026-04-28T03:00:00.000Z 值: 40
  5. 时间: 2026-04-28T04:00:00.000Z 值: 40

PaO2 数据（无）：
  没有找到

下一次血气检查时间：2026-04-28T08:33:00.000Z（约 8.5 小时后）
```
